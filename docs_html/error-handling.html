Error Handling

Sealed exception hierarchy

syncitron uses a sealed exception hierarchy rooted at syncitronException. You can pattern-match on specific types for granular error handling.

Exception Hierarchy

syncitronException  (sealed)
  ├── SyncNetworkException       — network / timeout / server error
  ├── SyncAuthException          — 401/403, session expired
  ├── ConflictResolutionException — custom resolver threw
  ├── SchemaMigrationException   — ALTER TABLE failed
  ├── LocalStoreException        — local DB read/write error
  ├── UnregisteredTableException — table not registered
  └── EngineConfigurationException — invalid config

Pattern Matching

try {
  await engine.syncAll();
} on SyncNetworkException catch (e) {
  if (e.isOffline) showOfflineBanner();
  else showError('Server error: ${e.statusCode}');
} on SyncAuthException catch (e) {
  redirectToLogin();
} on ConflictResolutionException catch (e) {
  log('Conflict on ${e.table} pk=${e.primaryKey}');
} on SchemaMigrationException catch (e) {
  reportFatalError(e);
} on LocalStoreException catch (e) {
  showError('Database error: ${e.message}');
} on syncitronException catch (e) {
  // catch-all for any syncitron error
  showError('Sync error: ${e.message}');
}

Exception Details

SyncNetworkException

FieldTypeDescription
tableStringThe table being synced
statusCodeint?HTTP status code, null if offline
isOfflineboolGetter — true when statusCode is null
causeObject?Original underlying exception

SyncAuthException

FieldTypeDescription
tableStringThe table being synced
messageStringDefault: 'Unauthorized. Session may have expired.'

ConflictResolutionException

FieldTypeDescription
tableStringTable where conflict occurred
primaryKeydynamicPrimary key of the conflicting record
causeObject?The error thrown by the custom resolver

SchemaMigrationException

FieldTypeDescription
tableStringTable being migrated
columnStringColumn that failed to add

LocalStoreException

FieldTypeDescription
tableStringTable with the local DB error
messageStringError description

UnregisteredTableException

Thrown when you attempt to sync a table that hasn't been registered via registerTable().

EngineConfigurationException

Thrown at setup time, e.g. using SyncStrategy.custom without a customResolver, or invalid config values.

ℹ️
All exceptions include a cause field for chaining the original underlying error.
