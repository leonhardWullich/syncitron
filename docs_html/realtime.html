Real-Time Sync

Event-driven sync with auto-reconnect

syncitron supports event-driven real-time synchronization for all four backends. When data changes on the server, the client syncs automatically — no polling needed.

Architecture

Remote Backend  ──▶  RealtimeSubscriptionProvider  ──▶  RealtimeSubscriptionManager
                         (per backend)                     (debounce → engine.syncTableByName())

Setup

final manager = RealtimeSubscriptionManager(
  config: RealtimeSubscriptionConfig.production(),
  provider: SupabaseRealtimeProvider(
    client: supabaseClient,
    postgresChangeEventAll: PostgresChangeEvent.all,
  ),
  engine: engine,
  logger: ConsoleLogger(),
);

await manager.initialize(['todos', 'projects']);

RealtimeSubscriptionConfig

ParameterTypeDefaultDescription
enabledbooltrueMaster switch
autoSyncbooltrueAuto-sync when changes detected
debounceDuration2 sDebounce window to coalesce rapid changes
connectionTimeoutDuration30 sConnection timeout
autoReconnectbooltrueAuto-reconnect on disconnect
maxReconnectAttemptsint5Max reconnect tries
backoffMultiplierdouble2.0Exponential backoff factor

Presets

RealtimeSubscriptionConfig.production()    // 2s debounce, 5 retries, 2× backoff
RealtimeSubscriptionConfig.development()   // 1s debounce, 10 retries, 1.5× backoff
RealtimeSubscriptionConfig.disabled()      // everything off

RealtimeSubscriptionManager API

MethodDescription

How It Works

  1. The provider subscribes to the backend's real-time channel for each table.
  2. When a RealtimeChangeEvent arrives (insert/update/delete), the table is added to the pending set.
  3. A debounce timer coalesces rapid changes (default 2 seconds).
  4. After the debounce window, engine.syncTableByName(table) is called for each pending table.
  5. On disconnect, auto-reconnect kicks in with exponential backoff.
