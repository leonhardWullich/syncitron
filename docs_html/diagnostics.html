Health & Diagnostics

Monitor sync infrastructure health

syncitron provides a diagnostics subsystem for monitoring the health of your sync infrastructure.

HealthCheckResult

HealthCheckResult(
  component: 'Database',
  status: HealthStatus.healthy,
  message: 'Connection is alive',
  details: {'tables': 5, 'total_rows': 1234},
)
FieldTypeDescription
componentStringName of the component
statusHealthStatushealthy / degraded / unhealthy
messageStringHuman-readable status message
detailsMap?Optional structured data
timestampDateTimeAuto-set to UTC now

HealthStatus Enum

ValueMeaning
healthyAll systems operational
degradedPartial issues (e.g. slow sync, outdated data)
unhealthyCritical failure

SystemHealth

Aggregates multiple HealthCheckResults:

final health = SystemHealth(results: {
  'db': dbCheck,
  'sync': syncCheck,
});

print(health.isHealthy);       // true if ALL healthy
print(health.overallStatus);   // worst status among all checks
print(health.toJson());        // structured JSON

Built-in Providers

DatabaseDiagnosticsProvider

Checks SQLite connectivity and returns table/row counts.

final dbDiag = DatabaseDiagnosticsProvider(database);
final result = await dbDiag.checkHealth();
final info = await dbDiag.getDiagnostics();
// {'database_type': 'SQLite', 'tables': 5, 'total_rows': 1234, ...}

SyncDiagnosticsProvider

Reports on the last sync status (time, success, metrics).

final syncDiag = SyncDiagnosticsProvider(
  lastSyncSuccessful: true,
  lastSyncTime: DateTime.now(),
  lastSyncMetrics: metrics.toJson(),
);

SystemDiagnosticsProvider

Combines multiple providers into a single health check:

final systemDiag = SystemDiagnosticsProvider([dbDiag, syncDiag]);
final health = await systemDiag.checkHealth();
// Overall status: worst of all sub-checks
