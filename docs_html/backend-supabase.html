Supabase Backend

PostgreSQL via Supabase + real-time WebSocket sync

SupabaseAdapter connects syncitron to a Supabase PostgreSQL backend. It supports cursor-based pagination, native batch upserts, and real-time subscriptions via WebSocket.

Setup

dependencies:
  syncitron: ^0.5.1
  supabase_flutter: ^2.12.0
  sqflite: ^2.4.2
import 'package:syncitron/syncitron.dart';
import 'package:supabase_flutter/supabase_flutter.dart';

final adapter = SupabaseAdapter(
  client: Supabase.instance.client,
  localStore: sqfliteStore,
  postgresChangeEventAll: PostgresChangeEvent.all,
  isAuthException: (e) => e is AuthException,
  updatedAtColumn: 'updated_at',  // default
);

Required Database Schema

Every table synced with syncitron must have these columns on Supabase:

CREATE TABLE todos (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  title TEXT NOT NULL,
  completed BOOLEAN DEFAULT false,
  -- Required by syncitron:
  updated_at TIMESTAMPTZ DEFAULT now(),
  deleted_at TIMESTAMPTZ NULL
);

-- Recommended: index for keyset pagination performance
CREATE INDEX idx_todos_updated_at ON todos(updated_at);
⚠️
The updated_at column must be auto-updated on every write. Use a Supabase trigger or set it in your application code.

Real-Time Subscriptions

final realtimeProvider = SupabaseRealtimeProvider(
  client: Supabase.instance.client,
  postgresChangeEventAll: PostgresChangeEvent.all,
  connectionTimeout: Duration(seconds: 30),
);

final manager = RealtimeSubscriptionManager(
  config: RealtimeSubscriptionConfig.production(),
  provider: realtimeProvider,
  engine: engine,
  logger: ConsoleLogger(),
);

await manager.initialize(['todos', 'projects']);

Batch Operations

SupabaseAdapter uses native PostgreSQL UPSERT for batch operations — true atomic writes for maximum throughput.

Security

  • Enable Row-Level Security (RLS) on all synced tables
  • Handle session expiry: catch SyncAuthException and re-authenticate
  • Never log auth tokens — use NoOpLogger in production if needed
