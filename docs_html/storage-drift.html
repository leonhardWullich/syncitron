Drift Storage

Type-safe SQL with compile-time safety

DriftStore wraps a Drift (typed SQL) database. Schema management is compile-time, so ensureSyncColumns() is a no-op — ensure your Drift schema includes is_synced and op_id columns.

Setup

dependencies:
  syncitron: ^0.5.1
  drift: ^2.14.0
  sqlite3_flutter_libs: ^0.5.0
import 'package:syncitron/syncitron.dart';

final store = DriftStore(
  tables: {'todos': todosTable, 'projects': projectsTable},
  readMetadataQuery: (table) => metadataDao.read(table),
  writeMetadataQuery: (table, json) => metadataDao.write(table, json),
  deleteMetadataQuery: (table) => metadataDao.delete(table),
);

Constructor

ParameterTypeDescription
tablesMap<String, dynamic>Map of table name → Drift table reference
readMetadataQueryFunction(String)Reads sync cursor JSON for a table
writeMetadataQueryFunction(String, String)Writes sync cursor JSON
deleteMetadataQueryFunction(String)Deletes sync cursor
isSyncedColumnStringColumn name (default: 'is_synced')
operationIdColumnStringColumn name (default: 'op_id')
⚠️
Your Drift schema must include is_synced (int) and op_id (text) columns. syncitron cannot add them at runtime.
