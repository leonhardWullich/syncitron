Configuration

syncitronConfig — presets and tuning

syncitronConfig is an immutable class controlling engine behaviour. Three factory constructors provide sensible presets.

Presets

syncitronConfig.production()   // large batches, aggressive retries
syncitronConfig.development()  // small batches, detailed logging
syncitronConfig.testing()      // minimal overhead, no logging/metrics

All Parameters

ParameterTypeDefaultDescription
batchSizeint500Max records per pull page / push batch
maxConcurrentSyncsint1Reserved for future parallel sync
operationTimeoutDuration30 sTimeout per remote operation
maxRetriesint3Max retry attempts on failure
initialRetryDelayDuration300 msFirst retry wait time
maxRetryDelayDuration30 sCap for exponential backoff
isSyncedColumnString'is_synced'Local column tracking sync status
operationIdColumnString'op_id'Local column for idempotency keys
autoSyncOnStartupboolfalseAuto-trigger syncAll after init
periodicSyncIntervalDuration?nullIf set, enables periodic sync timer
enableDetailedLoggingboolfalseLog individual record operations
collectMetricsbooltrueWhether to record sync metrics
validateOnCreationbooltrueRun validate() in constructor

Custom Configuration

final config = syncitronConfig(
  batchSize: 1000,
  maxRetries: 5,
  initialRetryDelay: Duration(seconds: 1),
  maxRetryDelay: Duration(minutes: 5),
  enableDetailedLogging: false,
  periodicSyncInterval: Duration(minutes: 10),
);

copyWith

Create a modified copy without mutating the original:

final custom = syncitronConfig.production().copyWith(
  batchSize: 2000,
  enableDetailedLogging: true,
);

Validation

validate() is called automatically during construction (unless validateOnCreation: false). It checks:

  • batchSize > 0
  • maxRetries >= 0
  • initialRetryDelay > 0
  • maxRetryDelay >= initialRetryDelay
