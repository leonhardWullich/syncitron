Multi-Engine (SyncManager)

Coordinate multiple SyncEngine instances

SyncManager coordinates multiple SyncEngine instances — useful for multi-tenant apps, workspaces, or isolated sync contexts.

Setup

final manager = SyncManager(
  logger: ConsoleLogger(),
  metricsCollector: InMemoryMetricsCollector(),
);

manager.registerEngine('workspace-a', engineA);
manager.registerEngine('workspace-b', engineB);

await manager.initializeAll();

API

MethodReturnsDescription
registerEngine(id, engine)voidRegister an engine with a unique ID
getEngine(id)SyncEngine?Retrieve an engine by ID
getAllEngines()List<SyncEngine>All registered engines
initializeAll()Future<void>Init all engines
syncAll()Future<Map<String, SyncSessionMetrics>>Sync all engines, returns metrics keyed by ID
syncEngine(id)Future<SyncSessionMetrics>Sync a specific engine by ID
startPeriodicSync(interval:)voidStart a periodic timer syncing all engines
stopPeriodicSync()voidStop the periodic timer
checkAllHealth()Future<Map<String, HealthCheckResult>>Health check all engines
dispose()Future<void>Stop timers, dispose all engines

Usage Pattern

// Sync all engines
final results = await manager.syncAll();
for (final entry in results.entries) {
  print('${entry.key}: ${entry.value.overallSuccess}');
}

// Periodic sync every 5 minutes
manager.startPeriodicSync(interval: Duration(minutes: 5));

// Cleanup
await manager.dispose();
