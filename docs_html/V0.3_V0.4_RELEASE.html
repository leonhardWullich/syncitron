New Feature: Custom Sync Strategies

Define domain-specific sync logic beyond the standard pull-push pattern.

Built-in Strategies

##### 1. StandardSyncStrategy (Default) Standard pull-push-conflicts pattern optimized for most use cases.

CODEBLOCK2

##### 2. OfflineFirstSyncStrategy Graceful degradation for unreliable networks. Tolerates multiple network errors and caches results for retry.

CODEBLOCK3

Best for: Apps in regions with iffy connectivity.

##### 3. ManualSyncStrategy Strict error handling with no automatic retry. Every error surfaces immediately.

CODEBLOCK4

Best for: Critical operations requiring explicit approval.

##### 4. PrioritySyncStrategy Syncs tables in configurable priority order. Critical tables fail-fast, optional tables tolerate errors.

CODEBLOCK5

Best for: Multi-tenant or complex applications with service dependencies.

##### 5. CompositeSyncStrategy Chains multiple strategies sequentially. Useful for pre/post-processing hooks.

CODEBLOCK6

Best for: Complex workflows requiring custom lifecycle management.

New UI Widgets (Sync Widgets)

1. SyncStatusWidget

Displays current sync status with manual sync button.

CODEBLOCK13

Shows:

  • "Ready" when idle
  • Sync progress indicator when syncing
  • "Sync completed successfully" on success

2. SyncMetricsCard

Card displaying sync metrics (pulled, pushed, duration, errors).

CODEBLOCK14

Displays:

  • Records pulled
  • Records pushed
  • Sync duration
  • Error count (if > 0)

3. SyncErrorBanner

Colored banner with context-aware error messages and retry/dismiss buttons.

CODEBLOCK15

Error categorization:

  • Network: Offline or unreachable
  • Auth: Session expired
  • Schema: Mismatch
  • Generic: Other errors
