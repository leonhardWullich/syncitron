Sync Orchestration

5 built-in strategies + custom orchestration

For advanced sync flows beyond the default pull-push pattern, syncitron provides SyncOrchestrationStrategy — a pluggable lifecycle for custom sync orchestrations.

How It Works

final metrics = await engine.syncWithOrchestration(
  OfflineFirstSyncOrchestration(),
);

The engine creates a SyncOrchestrationContext and calls:

  1. strategy.beforeSync(context) — setup, logging, pre-checks
  2. strategy.execute(context) — the actual sync logic (must return SyncSessionMetrics)
  3. strategy.afterSync(context, metrics) — cleanup, notifications

SyncOrchestrationContext

The context provides controlled access to engine internals:

MemberDescription
loggerThe engine's Logger
metricsCollectorThe engine's MetricsCollector
tableNamesList of registered table names
startTimeWhen the sync started
managedSyncTable(name)Sync a single table by name, returns SyncMetrics
managedSyncAll()Sync all tables, returns SyncSessionMetrics
shouldContinue()Returns false if cancelled
cancel()Cancels the orchestration

Built-in Strategies

StandardSyncOrchestration

Default pull-push for all tables. Same as syncAll().

OfflineFirstSyncOrchestration

Graceful degradation: continues syncing remaining tables even after network errors. Stops after maxNetworkErrors consecutive failures.

OfflineFirstSyncOrchestration(maxNetworkErrors: 3)

StrictManualOrchestration

Strict error handling — aborts on first failure. Useful for critical data.

PrioritySyncOrchestration

Syncs tables in priority order (higher number = synced first).

PrioritySyncOrchestration({
  'user_settings': 100,   // synced first
  'todos': 50,
  'analytics': 10,        // synced last
})

CompositeSyncOrchestration

Chains multiple strategies sequentially.

CompositeSyncOrchestration([
  PrioritySyncOrchestration({'settings': 100}),
  StandardSyncOrchestration(),
])

Custom Strategy

class MyStrategy extends SyncOrchestrationStrategy {
  @override
  Future<SyncSessionMetrics> execute(SyncOrchestrationContext ctx) async {
    final session = SyncSessionMetrics();

    // Sync critical tables first
    for (final table in ['users', 'settings']) {
      final m = await ctx.managedSyncTable(table);
      session.addTableMetrics(m);
      if (!ctx.shouldContinue()) break;
    }

    // Then sync the rest
    for (final table in ctx.tableNames) {
      if (['users', 'settings'].contains(table)) continue;
      final m = await ctx.managedSyncTable(table);
      session.addTableMetrics(m);
    }

    session.endTime = DateTime.now().toUtc();
    return session;
  }
}
