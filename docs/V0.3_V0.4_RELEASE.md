# Replicore v0.3.0 & v0.4.0 Release Guide

## Overview

**v0.3.0** focuses on removing deprecated APIs and introducing **Custom Sync Strategies** for domain-specific sync logic.

**v0.4.0** enhances **Multi-Engine Management** and provides **UI Helper Widgets** for seamless Flutter integration.

---

## v0.3.0: Custom Sync Strategies & API Cleanup

### Breaking Changes

#### 1. Removed Deprecated `onLog` Callback

**v0.2.0** (Deprecated):
```dart
SyncEngine(
  localStore: store,
  remoteAdapter: adapter,
  config: config,
  onLog: (msg) => print(msg), // ❌ Removed in v0.3.0
)
```

**v0.3.0** (Required):
```dart
SyncEngine(
  localStore: store,
  remoteAdapter: adapter,
  config: config,
  logger: ConsoleLogger(), // ✅ Use Logger interface
)
```

### New Feature: Custom Sync Strategies

Define domain-specific sync logic beyond the standard pull-push pattern.

#### Built-in Strategies

##### 1. **StandardSyncStrategy** (Default)
Standard pull-push-conflicts pattern optimized for most use cases.

```dart
final metrics = await engine.syncWithOrchestration(
  StandardSyncStrategy(),
);
```

##### 2. **OfflineFirstSyncStrategy**
Graceful degradation for unreliable networks. Tolerates multiple network errors and caches results for retry.

```dart
final metrics = await engine.syncWithOrchestration(
  OfflineFirstSyncStrategy(maxNetworkErrors: 3),
);
```

Best for: Apps in regions with iffy connectivity.

##### 3. **ManualSyncStrategy**
Strict error handling with no automatic retry. Every error surfaces immediately.

```dart
final metrics = await engine.syncWithOrchestration(
  ManualSyncStrategy(),
);
```

Best for: Critical operations requiring explicit approval.

##### 4. **PrioritySyncStrategy**
Syncs tables in configurable priority order. Critical tables fail-fast, optional tables tolerate errors.

```dart
final strategy = PrioritySyncStrategy({
  'subscriptions': 100, // Critical: fail-fast
  'user_settings': 50,
  'analytics': 10, // Optional: tolerate errors
});

final metrics = await engine.syncWithOrchestration(strategy);
```

Best for: Multi-tenant or complex applications with service dependencies.

##### 5. **CompositeSyncStrategy**
Chains multiple strategies sequentially. Useful for pre/post-processing hooks.

```dart
final composed = CompositeSyncStrategy([
  PreValidationStrategy(),
  StandardSyncStrategy(),
  PostCacheStrategy(),
]);

final metrics = await engine.syncWithOrchestration(composed);
```

Best for: Complex workflows requiring custom lifecycle management.

#### Creating Custom Strategies

Extend `SyncOrchestrationStrategy` to implement domain-specific logic.

**Example: Batch-Based Sync**
```dart
class BatchSyncStrategy extends SyncOrchestrationStrategy {
  final int batchSize;

  BatchSyncStrategy(this.batchSize);

  @override
  Future<SyncSessionMetrics> execute(SyncStrategyContext context) async {
    final metrics = SyncSessionMetrics();

    for (final tableName in context.tableNames) {
      if (!context.shouldContinue()) break;

      context.logger.info('Syncing batches for $tableName');
      
      try {
        final tableMetrics = await context.managedSyncTable(tableName);
        metrics.addTableMetrics(tableMetrics);
      } on SyncNetworkException {
        context.logger.warning('Network error, cancelling remaining batches');
        context.cancel();
      }

      // Custom: Delay between batches for rate limiting
      await Future.delayed(const Duration(milliseconds: 500));
    }

    return metrics;
  }

  @override
  Future<void> beforeSync(SyncStrategyContext context) async {
    context.logger.info('Pre-sync validation');
    // Custom validation logic
  }

  @override
  Future<void> afterSync(
    SyncStrategyContext context,
    SyncSessionMetrics metrics,
  ) async {
    context.logger.info('Sync complete, metrics: ${metrics.toJson()}');
    // Custom cleanup or analytics
  }
}

// Usage
final metrics = await engine.syncWithOrchestration(BatchSyncStrategy(50));
```

**Example: Health-Check-First Sync**
```dart
class HealthCheckFirstStrategy extends SyncOrchestrationStrategy {
  @override
  Future<SyncSessionMetrics> execute(SyncStrategyContext context) async {
    // Verify local DB health before sync
    context.logger.info('Running pre-sync health check');
    // ... health check logic ...

    return context.managedSyncAll();
  }
}
```

### SyncStrategyContext API

Provides controlled access to sync operations:

```dart
abstract class SyncStrategyContext {
  Logger get logger;
  MetricsCollector get metricsCollector;
  List<String> get tableNames;
  DateTime get startTime;

  // Sync operations
  Future<SyncMetrics> managedSyncTable(String tableName);
  Future<SyncSessionMetrics> managedSyncAll();

  // Lifecycle control
  bool shouldContinue();
  void cancel();
}
```

## v0.4.0: Enhanced Multi-Engine Management & UI Helpers

### Enhanced SyncManager

Coordinate multiple `SyncEngine` instances (e.g., multi-tenant apps).

#### Per-Engine Status Tracking

```dart
final manager = SyncManager();

// Register and name engines
manager.register('user_data', userEngine);
manager.register('analytics', analyticsEngine);

// Sync all engines with coordinated metrics
final aggregatedMetrics = await manager.syncAll();

print('User data metrics: ${aggregatedMetrics.forEngine('user_data')}');
print('Analytics metrics: ${aggregatedMetrics.forEngine('analytics')}');
```

#### Shared Logger & Metrics

```dart
final logger = ConsoleLogger();
final metrics = InMemoryMetricsCollector();

final engine1 = SyncEngine(
  localStore: store1,
  remoteAdapter: adapter1,
  logger: logger,
  metricsCollector: metrics,
);

final engine2 = SyncEngine(
  localStore: store2,
  remoteAdapter: adapter2,
  logger: logger,
  metricsCollector: metrics,
);

// Both engines share same logger and metrics collector
final manager = SyncManager()
  ..register('engine1', engine1)
  ..register('engine2', engine2);
```

#### Health Aggregation

```dart
final health = await manager.systemHealth();
print('Overall health: ${health.overallStatus}');
print('Per-engine health: ${health.engineStatuses}');
```

### New UI Widgets (Sync Widgets)

#### 1. **SyncStatusWidget**
Displays current sync status with manual sync button.

```dart
SyncStatusWidget(
  statusStream: engine.statusStream,
  onSync: () => SyncService.instance.sync(),
  showProgress: true,
)
```

Shows:
- "Ready" when idle
- Sync progress indicator when syncing
- "Sync completed successfully" on success

#### 2. **SyncMetricsCard**
Card displaying sync metrics (pulled, pushed, duration, errors).

```dart
SyncMetricsCard(
  metrics: lastSessionMetrics,
  elevation: 2,
  showWhenEmpty: true,
)
```

Displays:
- Records pulled
- Records pushed
- Sync duration
- Error count (if > 0)

#### 3. **SyncErrorBanner**
Colored banner with context-aware error messages and retry/dismiss buttons.

```dart
SyncErrorBanner(
  error: lastError,
  onRetry: () => SyncService.instance.sync(),
  onDismiss: () => setState(() => lastError = null),
)
```

Error categorization:
- **Network**: Offline or unreachable
- **Auth**: Session expired
- **Schema**: Mismatch
- **Generic**: Other errors

#### 4. **OfflineIndicator**
Small chip indicating offline status.

```dart
OfflineIndicator(
  isOnline: connectivity.isOnline,
  offlineLabel: 'Offline Mode',
)
```

Only visible when offline.

#### 5. **SyncButton**
Filled button with automatic loading state during sync.

```dart
SyncButton(
  onPressed: () => SyncService.instance.sync(),
  isSyncing: syncService.isSyncing,
  label: 'Sync Data',
  icon: Icons.cloud_sync,
)
```

Disables itself during sync, shows spinner.

#### 6. **SyncStatusPanel**
Comprehensive panel combining status, metrics, errors, and button.

```dart
SyncStatusPanel(
  statusStream: engine.statusStream,
  metrics: lastMetrics,
  error: lastError,
  onSync: () => SyncService.instance.sync(),
  onErrorDismiss: () => setState(() => lastError = null),
  showMetrics: true,
  showButton: true,
)
```

Ideal for dashboard or settings screens.

### Using UI Widgets Together

**Complete example:**
```dart
class SyncDashboard extends StatefulWidget {
  final SyncEngine engine;

  @override
  State<SyncDashboard> createState() => _SyncDashboardState();
}

class _SyncDashboardState extends State<SyncDashboard> {
  SyncSessionMetrics? _lastMetrics;
  ReplicoreException? _lastError;

  @override
  void initState() {
    super.initState();
    _setupSyncService();
  }

  void _setupSyncService() {
    SyncService.instance.syncError.addListener(() {
      setState(() => _lastError = SyncService.instance.syncError.value);
    });
  }

  @override
  Widget build(BuildContext context) {
    return SingleChildScrollView(
      child: Padding(
        padding: const EdgeInsets.all(16),
        child: Column(
          children: [
            // Status display
            SyncStatusWidget(
              statusStream: widget.engine.statusStream,
              onSync: () => SyncService.instance.sync(),
            ),
            
            const SizedBox(height: 16),
            
            // Error display
            SyncErrorBanner(
              error: _lastError,
              onRetry: () => SyncService.instance.sync(),
              onDismiss: () => setState(() => _lastError = null),
            ),
            
            const SizedBox(height: 16),
            
            // Metrics card
            if (_lastMetrics != null)
              SyncMetricsCard(metrics: _lastMetrics!),
            
            const SizedBox(height: 24),
            
            // Manual sync button
            SyncButton(
              onPressed: () async {
                final metrics = await widget.engine.syncAll();
                setState(() => _lastMetrics = metrics);
              },
              isSyncing: false,
              label: 'Manual Sync',
            ),
          ],
        ),
      ),
    );
  }
}
```

## Migration Guide: v0.2.0 → v0.3.0

### Remove `onLog` Callback

**Before:**
```dart
SyncEngine(
  ...
  onLog: (msg) => logger.log(msg), // ❌
)
```

**After:**
```dart
SyncEngine(
  ...
  logger: ConsoleLogger(), // ✅
)
```

### Update Custom Sync Logic

If you have custom onLog callbacks that dispatch metrics or errors, migrate to custom strategies:

**Before:**
```dart
SyncEngine(
  ...
  onLog: (msg) {
    if (msg.contains('error')) analyticsLog('sync_error', msg);
  }
)
```

**After:**
```dart
class AnalyticsTrackingStrategy extends SyncOrchestrationStrategy {
  @override
  Future<SyncSessionMetrics> execute(SyncStrategyContext context) async {
    return context.managedSyncAll();
  }

  @override
  Future<void> afterSync(
    SyncStrategyContext context,
    SyncSessionMetrics metrics,
  ) async {
    if (metrics.totalErrors > 0) {
      analyticsLog('sync_error', metrics.toJson());
    }
  }
}

await engine.syncWithOrchestration(AnalyticsTrackingStrategy());
```

## Migration Guide: v0.3.0 → v0.4.0

### Update UI Integration

**Before (no built-in widgets):**
```dart
ValueListenableBuilder<String>(
  valueListenable: SyncService.instance.syncStatus,
  builder: (_, status, __) => Text(status),
)
```

**After (using UI widgets):**
```dart
SyncStatusWidget(
  statusStream: engine.statusStream,
  onSync: () => SyncService.instance.sync(),
)
```

### Leverage New Widgets

Start using the new widget library to reduce boilerplate:

```dart
// Old: Manual error handling
if (error != null) {
  ScaffoldMessenger.of(context).showSnackBar(...);
}

// New: Automatic error banner
SyncErrorBanner(error: error)
```
